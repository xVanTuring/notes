# Podman & Docker

## First Web App
``` js
// index.js
const express = require("express")

const app = express()

app.get('/', (req, res) => {
    res.send('Hello World')
})
app.listen(8080, () => {
    console.log("listening on port 8080")
})
``` 
``` jsonc
// package.json
{
    "dependencies": {
        "express": "*"
    },
    "scripts": {
        "start": "node index.js"
    }
}
```
``` Dockerfile
FROM node:alpine
WORKDIR usr/app
COPY ./ ./
RUN npm install
CMD ["npm", "start"]
```
```bash
podman build . # 构建镜像
podman images  # 列出所有本地镜像
podman run -p <image-id> 8080:8080  # 运行镜像，绑定容器 8080 到本机 8080 端口
```
## Command List
* podman build . 构建image
* podman images  列出所有本地镜像
* podman run `<iamge_id>` 启动镜像
* podman ps -a 列出所有容器
* podman stop `<container_id>` 停止容器
* podman image rm `<image_id>`  删除镜像 (podman rmi `<image_id>`)
* podman container rm  `<container_id>`  删除容器 (podman rm `<container_id>`)
* podman logs -l 打印最新的一个容器的日志 (-l, --latest)
* podman top -l 打印最新一个容器的进程
* podman stop -l, podman rm -l 停止，删除最近一个容器

### run 
> podman run --help (man podman --help)
#### Synopsis
> podman run [options] image [command [arg ...]]
> podman container run [options] image [command [arg ...]]

#### Description
> podman run starts a process with its own file system, its own networking, and its own isolated process tree. 
#### Options
* podman run busybox ls -la
* --detach, -d=true|false
  > Detached Mode: run the container in the backgournd and print the new container ID. `false` by default. 
  > Reattach by `podman attach <container-id>`
  > `escape` or detach from the tty mode, use preconfigured keys `ctrl-p, ctrl-q`
* --tty, -t=true|false
  > Allocate a pseudo-TTY. When set to true, Podman will allocate a pseudo-tty and attach to the standard input of the container. This can be used, for example, to run a throwaway interactive shell. The default is false.
  > ``` bash
  > podman run -dt busybox
  > podman attach -l  
  > # ctrl-p ctrl-q to escape, exit command will stop the 
  > podman run -t busybox # can't type anything
  > podman stop -l # to stop
  > ##################
  > podman run -t busybox       # run with tty (terminal 1, can't type anything) 
  > podman attach -l            # attach to it (terminal 2)
  > ls -la                      # run command in terminal 2
  > # we will have the same result in terminal 1
  > ```
* --interactive, -i=true|false
  > When set to true, keep stdin open even if not attached. The default is false.
  >``` bash
  > podman run -t -i busybox
  > #######################
  > podman run -idt busybox # detached 
  > podman attach -l
  >``` 
* --publish, -p=ip:hostPort:containerPort | ip::containerPort | hostPort:containerPort | containerPort
  > Publish a container’s port, or range of ports, to the host.
  > `podman run -p 127.0.0.1::80` will have random port on host. use `podman port -l 80` to find out
* --publish-all, -P=true|false
  > Publish all exposed ports to random ports on the host interfaces. The default is false.
* --rm 
  > Automatically remove the container when it exits. The default is false.
* --restart=policy
  > Restart policy to follow when containers exit.
  > * no
  > * on-failure[:max_retries]
  > * always
  > * unless-stopped: Identical to always
  > Please note that restart will not restart containers after a system reboot. If this functionality is required in your environment, you can invoke Podman from a systemd.unit(5) file, or create an init script for whichever init system is in use. To generate systemd unit files, please see podman generate systemd.
* --workdir, -w=dir
  > Working directory inside the container. (`/` by default)
* --volume, -v[=[[SOURCE-VOLUME|HOST-DIR:]CONTAINER-DIR[:OPTIONS]]]
  > Create a bind mount. [detail](http://docs.podman.io/en/latest/markdown/podman-run.1.html#volume-v-source-volume-host-dir-container-dir-options)
* --name=name
  > Assign a name to the container.

### attach
#### Synopsis
> podman attach [options] container
> podman container attach [options] container
#### Description
> podman attach attaches to a running container using the container’s name or ID, to either view its ongoing output or to control it interactively.The container can detached from (and leave it running) using a configurable key sequence.
#### Options
* --no-stdin
  > Do not attach STDIN. The default is false.

### exec 
#### Synopsis
> podman exec [options] container [command [arg …]]
> podman container exec [options] container [command [arg …]]
#### Description
> podman exec executes a command in a running container.
#### Options
* --detach, -d
* --interactive, -i=true|false
* --tty, -t
* --workdir, -w=path
#### Exmaple
``` bash
podman run -t busybox           # terminal 0
podman exec -l ls               # terminal 1, print result of `ls`
podman exec -l -it sh           # terminal 2, type any command 
```

### start
#### Synopsis
> podman start [options] container …
> podman container start [options] container …

#### Example
``` bash
podman run -it busybox  # then exit
podman start -l         # start it again
podman attach -l        # attach to it or
podman start -a -l
```

### Bind local dir to container 
``` bash
podman run -d -v ./stocks/:/mnt --name mongo1 docker.io/mongo
podman exec -it mongo1 bash
```

### mongodbimport
```bash 
mongoimport --db catalog --collection books --type json --drop --file catalog.books.json
```
> 2021-09-11T10:15:36.040+0000	error connecting to host: could not connect to server: connection() error occured during connection handshake: dial tcp: i/o timeout
Fix by specifying the host: 
```
mongoimport --host 127.0.0.1 --db catalog --collection books --type json --drop --file catalog.books.json
```
